About the project
Intro

Our visualization framework enables a program analyst to understand inputs generated as part of a program testing campaign (also called fuzzing). The visualization is divided into two main sections:

  • Program Overview
  • Function Details

Program Overview

The Program Overview section shows a treemap of the functions composing the analyzed binary. Each function is represented as a square whose area depends on the size, expressed in number of basic code blocks, of the function.
Interaction
Users can interact with the treemap in different ways:

  • Hovering over each square will activate a tooltip showing the function name, the coverage during the current input, and the index of the current input.
  • Clicking on a square
selecting a function by clicking

Function Details

The Function Details section shows the layout of the selected function in terms of its basic blocks. It is represented as a directed graph, where each block is a node identified by its hexadecimal memory address.
